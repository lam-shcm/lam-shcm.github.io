# 个人主页程序编写说明

## 一、项目概述

本项目是一个以个人经历和真实作品为核心的响应式个人主页。页面围绕“音乐科技 × 音频工程 × 心理与认知 × 人工智能”建立个人定位，并通过代表项目、实习经历、教育背景、技能工具和音频试听展示个人实践。

项目使用 HTML、CSS、JavaScript 独立完成，未使用网页模板和前端组件库。视觉系统、页面结构、动画逻辑和交互组件均围绕个人专业方向设计。

## 二、设计思路

### 1. 信息架构

主页按照访客的阅读顺序组织为五个部分：

1. 首屏：快速说明个人身份、专业方向和当前状态。
2. 关于：解释跨学科背景及研究、工程、AI 产品三个实践维度。
3. 项目：通过四个代表案例展示不同类型的工作。
4. 经历：呈现实习、教育和技术工具。
5. 联系：提供公开邮箱和合作邀请。

项目材料较多，因此页面重点展示能够体现个人能力差异的四个案例：

- “人对非线性失真的感知实验”体现音频工程、人因实验和研究设计；
- “华为‘音悦家’”体现乐器与演奏技法调研、拟真乐器 UX 建模协作、混音滤镜与工程模板设计；
- “‘声栖’”体现 EEG 状态解析、音乐规则转译、生成式音频、TTS、自然环境声与自动混音；
- MORE EXPLORATIONS 中的疗愈音乐研究体现音乐心理、声学分析与跨学科文献研究。

### 2. 视觉语言

页面采用深色背景、荧光绿色信号线、暖橙色强调和蓝色技术辅助色。视觉灵感来自录音棚设备、示波器、频谱图和独立杂志排版。

项目图片直接使用真实项目材料，个人肖像来自简历。这样可以提升信息可信度，同时避免通用作品集模板的同质化效果。

### 3. 隐私与公开范围

页面只公开姓名和电子邮箱。手机号、出生日期、性别、家庭信息等内容没有写入网页代码。涉及保密项目时，页面仅保留简历中已有的概括性描述。

## 三、技术实现

### 1. HTML 语义结构

页面使用 `header`、`nav`、`main`、`section`、`article`、`aside`、`dialog` 等语义标签。标题层级从 `h1` 到 `h3` 逐级组织，图片均提供替代文字，按钮设置了对应的无障碍标签。

在移动端，导航按钮通过 `aria-expanded` 表示菜单状态；项目详情使用原生 `dialog` 元素，支持键盘 Escape 键关闭。

### 2. CSS 响应式布局

整体布局主要使用 CSS Grid 和 Flexbox：

- 首屏在桌面端采用双栏布局，移动端切换为单栏；
- 项目区桌面端为双列布局，重点项目横跨整行；
- 经历区由时间线与工具箱组成；
- 小屏幕下所有内容按阅读顺序重新排列。

字体尺寸使用 `clamp()` 根据屏幕宽度平滑变化。颜色、间距和页面宽度通过 CSS 自定义属性统一管理，便于后续修改。

### 3. 背景信号动画

`src/app.js` 使用 Canvas 2D API 绘制三组连续曲线。每条曲线由正弦和余弦函数叠加生成，鼠标位置会改变振幅、相位和纵向位置，从而形成与浏览者动作相关的“声音信号场”。

绘制过程使用 `requestAnimationFrame()` 与浏览器刷新节奏同步。设备像素比被限制在 2 以内，以平衡高清显示和性能。当系统开启“减少动态效果”时，Canvas 动画不会启动。

### 4. 滚动出现与导航状态

页面使用 `IntersectionObserver` 监听内容是否进入视口。元素第一次出现时添加 `is-visible` 类，CSS 随后完成透明度和位移动画。

滚动过程中，JavaScript 根据页面位置更新导航高亮，并在顶部导航离开首屏后增加半透明背景和模糊效果。

### 5. 项目详情弹窗

四个项目的详细信息存储在 `projectData` 对象中。点击“查看项目详情”后，程序根据项目编号填充标题、简介、个人工作、使用方法和项目结果，再调用 `showModal()` 打开弹窗。

这种数据与界面分离的方式减少了重复 HTML，增加新项目时只需添加一组数据和入口按钮。

### 6. 自定义音频播放器

AI 村歌项目卡片包含一段 35 秒试听。播放器使用原生 `audio` 元素提供音频能力，JavaScript 管理播放、暂停、时间显示与按钮状态。播放时，CSS 通过不同延迟的缩放动画模拟电平跳动。

### 7. 其他交互

- 移动端导航支持打开、关闭和点击链接后自动收起；
- “复制邮箱”按钮优先调用 Clipboard API，并提供兼容回退方案；
- 页面自动显示当前年份；
- 动画均兼容 `prefers-reduced-motion` 设置。

## 五、运行方法

电脑需要安装 Node.js。进入项目目录后执行：

```bash
npm install
npm run dev
```

用于提交或部署时执行：

```bash
npm run build
```

生成的 `dist` 文件夹包含完整静态网站，可以上传到静态网页托管平台。
