# 罗安民个人主页程序编写说明

## 一、作业与项目基本信息

- 作业方向：选项二“网页编程应用程序”
- 项目名称：罗安民个人主页
- 姓名：罗安民
- 学校：上海音乐学院
- 主要技术：HTML、CSS、JavaScript、Canvas API、Vite
- 在线地址：https://lam-shcm.github.io/
- 源码地址：https://github.com/lam-shcm/lam-shcm.github.io

本项目是一个围绕音乐科技、音频工程、心理与认知、AI 音乐产品方向设计的响应式
个人主页。网站把教育背景、专业能力、代表项目、实践经历和联系信息组织为一条连续
的阅读路径，使老师、招聘者或项目合作者能够在较短时间内理解个人定位，并继续查看
具体项目的方法、工具与成果。

项目未套用现成网页模板，也未使用 React、Vue 等前端框架。页面结构、视觉语言、
项目卡片、弹窗、动态背景和响应式适配均在原生 HTML、CSS、JavaScript 基础上实现。
Vite 只负责本地开发服务器和生产构建。

## 二、需求分析与设计目标

### 1. 内容需求

个人材料来自多份简历、项目 PPT 和研究经历，信息量较大。如果直接按传统简历顺序
堆叠，容易出现文字密集、项目关系不清的问题。因此网站将内容归纳为五类：

1. 身份与方向：姓名、学校、专业、所在地和关注方向。
2. 教育与荣誉：本科、辅修、微专业及创新实践经历。
3. 能力框架：研究声音、构建声音、连接智能。
4. 代表项目：三个可以体现研究、产品和交互能力的项目。
5. 实践与联系：实习经历、工具箱和公开联系邮箱。

### 2. 使用场景

网站主要考虑以下访问方式：

- 老师通过 GitHub 或提交材料检查源代码与实现说明；
- 招聘者通过公开链接快速浏览教育背景和代表项目；
- 手机用户通过纵向滚动阅读完整内容；
- 项目合作者通过弹窗了解个人承担的工作和使用工具；
- 访问者通过页面按钮复制邮箱或直接发送邮件。

### 3. 设计目标

- 信息清晰：教育背景靠前展示，重点项目有明显层级。
- 体现专业：视觉语言与声音、波形、录音设备和频谱概念相关。
- 保持可读：长文本控制行宽，标题、正文和辅助信息形成稳定层级。
- 支持交互：项目详情、滚动动画、导航状态和邮箱复制均有反馈。
- 适配多端：桌面、平板和手机使用同一套内容，布局根据宽度变化。
- 便于维护：项目数据集中管理，颜色和间距使用 CSS 变量控制。
- 可以部署：每次推送到 GitHub 后自动构建并发布在线版本。

## 五、HTML 结构实现

### 1. 语义标签

页面主要使用 `header`、`nav`、`main`、`section`、`article`、`aside`、`dialog`
等语义标签。每个主要区域通过 `id` 与标题的 `aria-labelledby` 建立对应关系，
有助于浏览器、搜索引擎和辅助技术理解页面结构。

### 2. 标题层级

- `h1` 只用于首屏核心标题；
- 每个主要区域使用 `h2`；
- 项目、能力卡片、经历和工具组使用 `h3`；
- 英文栏目名、编号和标签作为辅助信息，不替代标题层级。

### 3. 图片与链接

项目图片均设置 `alt` 替代文字。华为项目链接使用 `target="_blank"` 打开新窗口，
并通过 `rel="noreferrer"` 减少来源信息传递。联系区域使用 `mailto:` 链接调用
本机邮件程序。

### 4. 无障碍基础

- 页面开头提供“跳到主要内容”链接，键盘聚焦时显示；
- 移动端菜单按钮使用 `aria-expanded` 表达开关状态；
- 项目按钮通过 `aria-haspopup="dialog"` 说明会打开弹窗；
- 弹窗使用原生 `dialog` 元素，可通过 Escape 键关闭；
- 状态提示使用 `role="status"` 和 `aria-live="polite"`；
- 纯装饰图形使用 `aria-hidden="true"`，避免干扰读屏。

## 六、CSS 视觉与响应式实现

### 1. 全局设计变量

`src/styles.css` 在 `:root` 中定义主要设计变量：

```css
--ink: #0c0d0f;
--paper: #f2f0e9;
--lime: #c8ff45;
--coral: #ff6b4a;
--blue: #75a7ff;
--header-height: 84px;
--page-pad: clamp(22px, 4.3vw, 72px);
--content-width: 1440px;
```

颜色、页边距、内容最大宽度和动画缓动统一由变量控制。修改变量即可同步调整多个
区域，减少重复数值。

### 2. 布局方法

- CSS Grid：首屏双栏、项目网格、经历与工具箱、教育经历四列；
- Flexbox：导航、按钮、标签、联系信息等一维排列；
- `clamp()`：让标题、正文和间距在不同宽度间平滑变化；
- `minmax()`：防止网格列过窄造成内容挤压；
- `object-fit` 与 `object-position`：控制不同尺寸项目图片的裁切方式。

### 3. 响应式断点

项目设置三个主要断点：

- 1100px 以下：重点项目由左右结构切换为上下结构，经历区变为单列；
- 820px 以下：导航变为全屏菜单，首屏变为单列，教育经历和项目变为单列；
- 560px 以下：进一步缩小标题，基本信息纵向排列，时间线、弹窗和联系按钮适配手机。

页面没有设置固定设备型号，而是根据内容是否拥挤来确定断点。因此在常见桌面、
平板和手机宽度下都能保持阅读顺序。

### 4. 视觉细节

- 固定导航在滚动后增加半透明背景和模糊效果；
- 卡片悬停时产生轻微上移和边框颜色变化；
- 项目图片悬停时进行小幅缩放；
- 深色背景上叠加低透明度噪声纹理，增强录音设备和独立杂志的质感；
- 首屏头像保留圆拱形轮廓，取消覆盖人物的网格，保证主体清晰。

## 七、JavaScript 功能实现

### 1. 移动端菜单

`setMenu(open)` 统一控制三个状态：

- 修改菜单按钮的 `aria-expanded`；
- 在导航上切换 `is-open` 类；
- 在 `body` 上切换 `menu-open`，防止菜单打开时背景继续滚动。

点击菜单按钮会反转状态，点击任意导航链接后菜单自动关闭。

### 2. 滚动导航状态

页面监听 `scroll` 事件，根据滚动距离为顶部导航添加 `is-scrolled`。程序使用
“当前滚动位置 + 视口高度的 34%”作为判断标记，依次检查各区域的 `offsetTop`，
并给当前导航链接添加 `is-active`。

滚动监听使用 `{ passive: true }`，避免监听函数阻塞浏览器默认滚动行为。

### 3. 内容进入动画

`IntersectionObserver` 负责监听带有 `.reveal` 的元素。当元素进入视口时：

1. 添加 `is-visible` 类；
2. CSS 完成透明度和位移动画；
3. 调用 `unobserve()` 停止继续监听，减少后续计算。

元素按索引获得 0—195ms 的分组延迟，使同一区域内容依次出现。

### 4. 数据驱动的项目弹窗

三个项目的详细信息集中保存在 `projectData` 对象中。每个项目包含：

- `eyebrow`：英文分类；
- `title`：项目名称；
- `summary`：项目背景；
- `role`：个人工作；
- `tools`：方法与工具；
- `outcome`：项目输出。

项目按钮通过 `data-project-open` 保存项目键名。点击按钮后，程序读取对应对象，
使用 `textContent` 填充标题和正文，并由 `fillList()` 动态创建列表项，最后调用
`dialog.showModal()`。这种结构避免在 HTML 中重复三套详情内容，同时降低使用
`innerHTML` 带来的不必要风险。

弹窗右上角按钮可关闭弹窗，点击弹窗外部的遮罩区域也会关闭。

### 5. 邮箱复制

`copyText()` 优先使用 Clipboard API：

```js
await navigator.clipboard.writeText(text);
```

如果浏览器拒绝剪贴板权限，程序会创建一个透明的临时 `textarea`，选中文本后调用
`document.execCommand("copy")` 作为兼容方案。复制成功或失败后，页面右下角显示
2.6 秒状态提示。

### 6. 自动年份

页面通过 `new Date().getFullYear()` 获取当前年份，并写入带有 `data-year` 的元素，
因此页脚年份无需每年手动修改。

### 7. Canvas 声音信号背景

`startSignalCanvas()` 在固定画布上绘制三条连续曲线。主要过程如下：

1. 根据窗口宽高与设备像素比调整 Canvas 实际尺寸；
2. 将设备像素比限制在 2 以内，兼顾清晰度与性能；
3. 记录鼠标位置并换算为 0—1 的比例；
4. 使用正弦和余弦函数叠加生成曲线；
5. 鼠标位置改变振幅、相位和纵向位置；
6. 使用 `requestAnimationFrame()` 与浏览器刷新同步绘制；
7. 窗口尺寸变化时重新计算画布。

指针当前位置通过缓慢插值接近目标位置，使动画变化更加平滑。三条曲线使用绿色、
蓝色和橙色，并设置不同频率、振幅和透明度，形成声音信号场。

如果操作系统开启“减少动态效果”，程序不会启动 Canvas 动画，CSS 也会将动画和
过渡时间压缩到最低。

## 八、构建与部署

### 1. 本地开发

电脑需要安装 Node.js。进入项目根目录后执行：

```bash
npm install
npm run dev
```

终端会显示本地地址，浏览器访问该地址即可查看开发版本。

### 2. 生产构建

```bash
npm run build
```

Vite 会完成以下工作：

- 合并和压缩 CSS、JavaScript；
- 为构建文件添加内容哈希，减少缓存冲突；
- 复制 `public/assets/` 中的静态素材；
- 在 `dist/` 中生成可直接部署的静态网站。

使用以下命令检查构建结果：

```bash
npm run preview
```

### 3. GitHub Pages 自动发布

仓库名为 `lam-shcm.github.io`。`.github/workflows/deploy.yml` 在 `main` 分支更新时
执行以下步骤：

1. 检出源码；
2. 安装 Node.js；
3. 使用 `npm ci` 按锁定版本安装依赖；
4. 执行 `npm run build`；
5. 上传 `dist/`；
6. 发布到 GitHub Pages。

因此公开地址保持为 https://lam-shcm.github.io/，后续修改只需提交并推送到
`main` 分支。
